#+title: openclaw.el
#+description: Emacs interface to OpenClaw
#+options: toc:2

* Overview

=openclaw.el= is an Emacs package that provides a complete interface to [[https://github.com/openclaw/openclaw][OpenClaw]], your personal AI assistant.

Each OpenClaw session becomes its own Emacs buffer, and you get:
- Leader-key based keybindings for all slash commands
- Direct slash command support for those who prefer it
- Full feature parity with the OpenClaw TUI
- Multiple simultaneous sessions in separate buffers

* Installation

** With straight.el and use-package (recommended)

If you use [[https://github.com/radian-software/straight.el][straight.el]] with [[https://github.com/jwiegley/use-package][use-package]], add this to your init file:

#+begin_src elisp
(use-package openclaw
  :straight (openclaw :type git :host github :repo "Kyvero-Vexus/openclaw.el")
  :config
  (setq openclaw-gateway-url "ws://127.0.0.1:18789"))
#+end_src

** With straight.el only

#+begin_src elisp
(straight-use-package
 '(openclaw :type git :host github :repo "Kyvero-Vexus/openclaw.el"))
#+end_src

** With Quelpa

#+begin_src elisp
(use-package openclaw
  :quelpa (openclaw :fetcher github :repo "Kyvero-Vexus/openclaw.el"))
#+end_src

** Manual installation

Clone this repository and add to your =load-path=:

#+begin_src elisp
(add-to-list 'load-path "/path/to/openclaw.el")
(require 'openclaw)
#+end_src

** Dependencies

- Emacs 27.1 or later
- [[https://github.com/ahyatt/emacs-websocket][websocket]] package (for WebSocket communication)

straight.el and package.el will install dependencies automatically.

* Quick Start

1. Connect to your OpenClaw gateway:

   #+begin_src elisp
   M-x openclaw-connect RET
   #+end_src

2. Start chatting:

   #+begin_src elisp
   M-x openclaw-chat RET
   #+end_src

3. Type your message and press =C-c C-c= to send.

* Configuration

** Gateway settings

#+begin_src elisp
(setq openclaw-gateway-url "ws://127.0.0.1:18789"
      openclaw-gateway-token "your-token-here")
#+end_src

** Leader key

By default, leader-style commands are accessible under =C-c= in =openclaw-mode=:

#+begin_src elisp
(setq openclaw-leader-key "C-c")  ; default
#+end_src

* Keybindings

** Chat mode (=openclaw-chat-mode=)

| Key     | Command                    |
|---------+----------------------------|
| =RET=     | Send message               |
| =C-c C-c= | Send message               |
| =C-c C-l= | List sessions              |
| =C-c C-s= | Switch session             |
| =C-c C-n= | New chat                   |
| =C-c C-h= | Show history               |
| =C-c C-q= | Close connection           |

** Leader keys

In =openclaw-mode=, single-letter commands are available:

| Key | Command             |
|-----+---------------------|
| =c=   | Connect             |
| =C=   | Close connection    |
| =l=   | List sessions       |
| =s=   | Switch session      |
| =n=   | New chat            |
| =m=   | Send message        |
| =h=   | Show history        |
| =?=   | Help                |

* Slash Commands

You can still use OpenClaw slash commands directly by typing them in the chat buffer:

- =/status= - Show session status
- =/clear= - Clear chat history
- =/model= - Change model
- =/help= - Show gateway help

* Gateway Protocol

=openclaw.el= communicates with the OpenClaw gateway via JSON-RPC over WebSocket:

- Default URL: =ws://127.0.0.1:18789=
- Protocol: JSON-RPC 2.0
- Methods used:
  - =sessions.list= - List available sessions
  - =chat.send= - Send a message
  - =chat.history= - Fetch message history

See the [[https://docs.openclaw.ai/gateway][OpenClaw Gateway documentation]] for more details.

* Troubleshooting

** Authentication failures with gateway v2026.3.7+

Starting with OpenClaw gateway v2026.3.7, the =gateway.auth.mode= configuration
field is required.  If you see authentication errors ("auth mode not configured"
or similar), ensure your =~/.openclaw/openclaw.json= includes:

#+begin_src json
{
  "gateway": {
    "auth": {
      "mode": "token",
      "token": "your-token-here"
    }
  }
}
#+end_src

Valid =mode= values: =token=, =password=, =none=.  When =mode= is omitted the
gateway will reject all connection attempts.
